Design Patterns

Course: CSE 3421 | Source: Gang of Four (GoF)

Standard solutions to common programming problems. They ensure loose coupling and make code flexible.

1. Creational Patterns

Focus: How objects are instantiated (created).

A. Factory Method

Defines an interface for creating an object, but lets subclasses decide which class to instantiate.

Example: A ShapeFactory that creates Circles or Squares based on input.

ShapeFactory
+ getShape(type): Shape
→
<<interface>>Shape
+ draw()
←
Circle
+ draw()
Rectangle
+ draw()

B. Abstract Factory

A "Factory of Factories". Provides an interface for creating families of related objects (e.g., Shapes AND Colors).

Analogy: A FactoryProducer gives you either a PaintFactory or an AnimationFactory.

C. Singleton

Ensures a class has only one instance and provides a global point of access to it.

Use Case: Database Connections, HTTP Connections.

DbConnection
- static instance
- private Constructor()
+ getInstance()

2. Structural Patterns

Focus: How classes and objects are combined/composed.

A. Adapter Pattern

Acts as a bridge between two incompatible interfaces.

Real World Example: Travel Adapter (US Plug into EU Socket).

Client
Wants to use Target
→
Adapter
Translates request
→
Adaptee
Incompatible Class

B. Decorator Pattern

Adds new functionality to an existing object without altering its structure. "Wraps" the object.

Example: Pizza Toppings. Start with Pizza, wrap it in CheeseDecorator, wrap that in PepperoniDecorator.

3. Behavioral Patterns

Focus: How objects communicate with each other.

A. State Pattern

Allows an object to alter its behavior when its internal state changes.

Example: A TV Remote.

  • If State is On, the power button turns it Off.
  • If State is Off, the power button turns it On.

B. Strategy Pattern

Defines a family of algorithms and makes them interchangeable at runtime.

Example: Sorting or Math Operations.

Context
executeStrategy()
→
<<interface>>Strategy
+ doAction()
AddStrategy
SubStrategy
MulStrategy

Practice Scenarios (For Exam)

  • Traffic Light: Use State Pattern (Red -> Green -> Yellow).
  • Audio Player (mp3, wav, vlc): Use Adapter Pattern to make incompatible formats play.
  • Sort Array (Bubble, Quick, Merge): Use Strategy Pattern to swap algorithms.
  • Coffee Shop Ingredients: Use Factory/Abstract Factory.
