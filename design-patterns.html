Design Patterns (GoF)

A standard solution to a common programming problem. Patterns ensure loose coupling and make code more flexible and reusable.

1. Creational Patterns

Concerned with how objects are instantiated (created).

A. Factory Method

Defines an interface for creating an object, but lets subclasses decide which class to instantiate.
Key Takeaway:
  • The client (Kiosk) doesn't know exactly what payment method is being created.
  • It asks the `PaymentFactory`, which returns a `Cash`, `CreditCard`, or `Coupon` object.

B. Abstract Factory

Provide an interface for creating families of related or dependent objects (Factory of Factories).
Key Takeaway:
  • Used when you have multiple families (e.g., Honda Civic parts vs. Honda Accord parts).
  • Ensures you don't mix parts (e.g., Putting a Civic Engine in an Accord).

C. Singleton

Ensures a class has only one instance and provides a global point of access to it.
Key Takeaway:
  • Note the private constructor - DbConnection().
  • The only way to get the object is the static method + getInstance().

2. Structural Patterns

Concerned with how classes and objects are composed (combined).

A. Adapter Pattern

Converts the interface of a class into another interface the clients expect. Bridges the gap between incompatible interfaces.
Key Takeaway:
  • The `Client` expects a Triangle, but the old `Calculator` only works with Rectangles.
  • The `CalculatorAdapter` sits in the middle, translating Triangle inputs into Rectangle inputs.

B. Decorator Pattern

Attaches additional responsibilities to an object dynamically. "Wraps" the object.
Key Takeaway:
  • Notice how `PatternDecorator` and `ColorDecorator` both extend Shape and contain a Shape.
  • This allows you to layer effects (e.g., a Red Circle with a Pattern).

3. Behavioral Patterns

Concerned with communication between objects.

A. State Pattern

Allows an object to alter its behavior when its internal state changes.
Key Takeaway:
  • The `Remote` behaves differently depending on whether it is in `OnState` or `OffState`.
  • Removes complex if/else statements.

B. Strategy Pattern

Defines a family of algorithms, encapsulates each one, and makes them interchangeable at runtime.
Key Takeaway:
  • The `Client` wants to sort, but doesn't care how.
  • You can swap `BubbleSort`, `QuickSort`, or `InsertionSort` without breaking the client code.

Practice Scenarios for Exam

  • Traffic Light: Use State Pattern (Red -> Green -> Yellow).
  • Audio Player (mp3, wav, vlc): Use Adapter Pattern to make incompatible formats play.
  • Sort Array: Use Strategy Pattern to swap algorithms.
  • Website Builder / Presentation Software: Use Decorator Pattern to add features (fonts, colors) gradually.
