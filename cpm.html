Critical Path Method (CPM)

Course: CSE 3421 | Topic: Scheduling & Estimation

1. Definitions

CPM is a project planning method used to determine the earliest time a project can be completed.

  • Critical Task: A task that cannot be delayed. If it is delayed, the whole project is delayed.
  • Critical Path: The longest path through the network. It determines the minimum time required for the project.
  • Float (Slack): How much an activity can be delayed without affecting the project deadline.
    Formula: Float = LS - ES (If Float is 0, it is a Critical Task).

Step C: Identifying the Critical Path

The Critical Path is the sequence where Float = 0 (ES == LS).

Calculated Floats:

  • Activity F: Start 0, Duration 10, Finish 10. (Must finish by 10 for G to start). Float = 0.
  • Activity G: Start 10, Duration 3, Finish 13. (Project ends at 13). Float = 0.
  • Activity H: Finishes at 11, but project allows until 13. Float = 2.

Final Critical Path

START → F → G → FINISH

Total Project Duration: 13 Weeks
